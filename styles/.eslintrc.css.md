# CSS style notes

Brief house style for everything under `styles/`.

## Formatting

- **Indent: 2 spaces.** No tabs. Configure your editor accordingly.
- One selector per line in multi-selector rules.
- One blank line between rule blocks; no blank lines inside a block.
- Lowercase hex (`#1a2433`, never `#1A2433`) — but prefer tokens.
- End every declaration with a semicolon, including the last one.

## Property order (top to bottom inside a block)

1. **Positioning** — `position`, `inset`, `top`, `z-index`
2. **Box** — `display`, `flex`/`grid`, `width`, `height`, `margin`,
   `padding`, `border`
3. **Text** — `font-*`, `line-height`, `letter-spacing`,
   `text-transform`, `color`
4. **Visual** — `background`, `box-shadow`, `opacity`, `transform`,
   `transition`, `animation`

## Colors

- **Token-only.** No raw hex in component or page modules; every color
  must come from a `--token` defined in `_tokens.css`. The only place
  raw hex is allowed is `_tokens.css` itself.
- Stick to the brand palette (`BRAND.md` §3). Don't invent shades.

## Imports

- New modules added to `main.css` go in the section that matches their
  layer (Foundations → Components → Pages). Never under Legacy.
- Page modules are imported in roughly the order pages are migrated.
