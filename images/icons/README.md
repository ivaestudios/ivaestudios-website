any logo or brand-color change and every file below will be rebuilt in place.

## File catalog

| File | Size | Purpose |
|---|---|---|
| `favicon.ico` | 16 / 24 / 32 / 48 multi-size | Universal browser tab icon |
| `favicon-16.png` | 16×16 | Legacy browsers |
| `favicon-32.png` | 32×32 | High-DPI tab icon |
| `favicon-48.png` | 48×48 | Windows tile, taskbar |
| `apple-touch-icon.png` | 180×180 | iOS home-screen (NO transparency — Apple flattens) |
| `icon-192.png` | 192×192 | Android home-screen, PWA |
| `icon-512.png` | 512×512 | PWA splash, large surfaces |
| `icon-maskable-512.png` | 512×512 | Adaptive icon — Æ at ~50% so circular masks never crop |
| `../og-default.jpg` | 1200×630 | Default Open Graph image (≤300 KB) |
| `../twitter-card.jpg` | 1200×675 | Default Twitter `summary_large_image` |

A copy of `favicon.ico` also lives at the **repository root** (`/favicon.ico`)
so requests to the conventional path `https://ivaestudios.com/favicon.ico`
resolve without a redirect. Both files are generated by the build script —
the root copy is a `shutil.copyfile`, not a symlink, because some hosts
mangle symlinks during deploy.

## HTML `<head>` snippet (copy-paste into every page)

```html
<link rel="icon" href="/favicon.ico" sizes="any">
<link rel="icon" href="/images/icons/favicon-32.png" sizes="32x32" type="image/png">
<link rel="apple-touch-icon" href="/images/icons/apple-touch-icon.png">
<link rel="manifest" href="/site.webmanifest">
<meta name="theme-color" content="#1A2433">
<meta property="og:image" content="https://ivaestudios.com/images/og-default.jpg">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:image" content="https://ivaestudios.com/images/twitter-card.jpg">
```

Per-page `<meta property="og:image">` overrides the default — set it on
gallery posts, blog posts, and any landing page that has its own hero.

## How the OG and Twitter cards were generated

`build_icons.py` composes the cards in PIL directly. Composition:

- **Background:** Deep Atlantic (`#0F1620`) with a faint radial vignette.
- **Eyebrow:** `EST. 2018  ·  CANCÚN`, Inter-substitute, 0.40em tracking, warm-sand.
- **Wordmark:** `IVAE` in DM Serif Display substitute (Times Bold), Resort White.
- **Lockup line:** `STUDIOS` in Inter substitute, 0.40em tracking, Resort White.
- **Hairline rule:** half-width gold (`#C4A35A`).
- **Accent:** `Luxury Resort Photography · Riviera Maya · Cancún · Los Cabos` in
  Bodoni Moda italic substitute, gold.

### Font substitutions

The brand fonts (DM Serif Display, Bodoni Moda, Inter) ship as **WOFF2**, which
PIL cannot load. The script maps to the closest local TTF/TTC available on
macOS: Bodoni 72 → italic serif; Helvetica Neue → Inter; Times New Roman Bold
→ DM Serif Display. The visual register (high-contrast serif + clean sans +
italic accent) is preserved.

If a future build is run on a Linux CI, install `fonts-bodoni`,
`fonts-cormorant-garamond` or convert the brand WOFF2 to TTF and update the
candidate list at the top of `build_icons.py`. Until then, regenerate locally
on a macOS workstation and commit the resulting binaries.

